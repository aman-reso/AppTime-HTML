In-App Prominent Disclosure Template

This template provides the required in-app prominent disclosure for Google Play compliance regarding Installed Application information collection.

⚠️ REQUIRED IN-APP PROMINENT DISCLOSURE

AppTime collects, transmits, syncs, and stores your Installed Application information (including application names, package identifiers, and usage statistics) to enable screen time tracking and app usage analytics features, when you use the app to monitor your device usage patterns.

Implementation Requirements

✅ Must-Have Requirements:

  • Display Location: Must be shown within the app itself, in normal usage flow (NOT in a menu or settings)
  • Timing: Must be displayed BEFORE any data collection begins
  • Standalone: Cannot be combined with other unrelated disclosures
  • Clear Language: Must use the exact format: "[App] collects/transmits/syncs/stores [data type] to enable [feature], [scenario]"
  • Affirmative Consent: Must require explicit user action (tap "Accept", check a box, etc.)
  • No Auto-Dismiss: Cannot use auto-dismissing or expiring messages
  • No Implied Consent: Navigation away (back button, home button) does NOT count as consent

Recommended Disclosure Text

Title: Data Collection Disclosure

Main Text:
AppTime collects, transmits, syncs, and stores your Installed Application information (including application names, package identifiers, and usage statistics) to enable screen time tracking and app usage analytics features, when you use the app to monitor your device usage patterns.

Additional Details (expandable):
• We collect information about all applications installed on your device
• This data is transmitted to our secure servers for cloud sync and analytics
• Your data is encrypted and stored securely
• You can opt-out of data transmission in settings at any time
• Learn more in our Privacy Policy

Consent Options:
[✓] I understand and consent to the collection of my Installed Application information

[Accept and Continue] [Learn More] [Decline]

UI Implementation Examples

Example 1: Full-Screen Modal (Recommended)

Display as a modal dialog that covers the entire screen, preventing interaction with the app until the user makes a choice.

Example 2: Prominent Banner

Display as a prominent banner at the top of the main screen that cannot be dismissed without user action.

Example 3: Dedicated Onboarding Screen

Show as the first screen after app installation, before any other functionality is available.

Consent Mechanism Requirements

The consent mechanism must:

  • Require explicit affirmative action (button tap, checkbox selection)
  • Be clear and unambiguous
  • Not allow proceeding without consent (if data collection is required)
  • Provide an option to decline (if possible) or clearly state that the app cannot function without consent

Code Implementation Checklist

Before Data Collection:

  • ✓ Check if user has already provided consent
  • ✓ If not, display prominent disclosure
  • ✓ Wait for explicit user consent
  • ✓ Store consent status securely
  • ✓ Only then begin collecting Installed Application information

Disclosure Display:

  • ✓ Show in main app flow (not in settings)
  • ✓ Cannot be dismissed by back/home button
  • ✓ Requires explicit action to proceed
  • ✓ Link to full Privacy Policy

Important Notes

  • This disclosure is REQUIRED in addition to the Privacy Policy - the Privacy Policy alone is not sufficient
  • The disclosure must be shown within the app, not just in a web page
  • Users must see this before any Installed Application information is collected
  • If a user declines, you must either disable data collection or prevent app usage (depending on your app's functionality)
  • Keep a record of user consent for compliance purposes

Testing Checklist

  • ✓ Disclosure appears before any data collection
  • ✓ Cannot proceed without explicit consent
  • ✓ Back/home button does not dismiss disclosure
  • ✓ Text clearly states what data is collected and why
  • ✓ Privacy Policy is easily accessible
  • ✓ User can opt-out later in settings
  • ✓ Consent status is properly stored and respected

This template should be implemented in your app's native code (Java/Kotlin for Android, Swift/Objective-C for iOS).
